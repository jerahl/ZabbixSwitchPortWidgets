ifHighSpeed is Zero for All Interfaces — Firmware Bug

ifHighSpeed returns 0 for every interface including eth0. This is an IQ Engine firmware limitation — the 64-bit ifHighSpeed object is not populated on the AP305C. The standard Zabbix template item net.if.speed[ifHighSpeed.N] will always return 0 and must NOT be used.

Use ifSpeed (OID .1.3.6.1.2.1.2.2.1.5.10) for link speed. The 32-bit Gauge correctly reports 1,000,000,000 (1 Gbps) for eth0. The 64-bit ifHCInOctets and ifHCOutOctets counters ARE correctly populated and must be used for all throughput items.

4 Per-Radio RF Config — .26928.1.1.1.2.1.5.1

Sub-table indexed by radio ifIndex. Provides channel, TX power, and channel width encoding per radio. All three are config-class values (change infrequently) rather than continuous time-series.

.26928.1.1.1.2.1.5.1.2.ifIndex 13 dBm 13 dBm TX power (dBm) SNMP walk discovery item — Radio card display + sparkline
.26928.1.1.1.2.1.5.1.3.ifIndex 160 160 Channel width (encoding TBD — see note) Validate against XIQ channel width for same AP
Both radios are on 5 GHz channels (36 and 157). The AP305C is a 2.4/5 GHz dual-radio AP, but this unit is configured in dual-5 GHz mode — both radios dedicated to 5 GHz. This is a common TCS configuration choice to reduce 2.4 GHz interference. The Wireless tab radio card layout must NOT assume radio 0 = 2.4 GHz. Label each card from the channel number: channels 1–14 → 2.4 GHz, channels ≥ 36 → 5 GHz.
Channel width value 160: The AP305C is 802.11ac Wave 1 and physically limited to 80 MHz channel width. The value 160 is Aerohive's internal encoding and does not mean 160 MHz. Validate by comparing the SNMP value against the channel width shown in XIQ for the same AP during M0 live validation.

6 Noise Floor — .26928.1.3.1.0 — Available with Preprocessing

The noise floor for both radios is exposed as a single string scalar:

.1.3.6.1.4.1.26928.1.3.1.0 = STRING: "wifi0: -96 dBm, wifi1: -96 dBm, "

This is a combined string for both radios. Zabbix cannot store a string as a numeric history item directly. The solution is a master item + two dependent items, each with a JS preprocessing step to extract the per-radio integer value:

// Dependent item preprocessing — noise floor wifi0
var m = value.match(/wifi0:\s*([-\d]+)\s*dBm/);
return m ? parseInt(m[1], 10) : null;

// Dependent item preprocessing — noise floor wifi1
var m = value.match(/wifi1:\s*([-\d]+)\s*dBm/);
return m ? parseInt(m[1], 10) : null;
The −96 dBm values in this walk reflect a lightly-loaded AP. This is a reasonable noise floor for 5 GHz. Real values during school hours will vary. Because Zabbix stores history via the master item poll, both radio noise floor sparklines are fully possible — something XIQ cannot provide.

The Overview tab was designed with four rings: CPU, Memory, PoE Draw, Temperature. With PoE and Temperature confirmed absent from all data sources, the layout must change. Proposed: replace with two confirmed SNMP rings and two RF signal badges sourced from the enterprise MIB noise floor.
