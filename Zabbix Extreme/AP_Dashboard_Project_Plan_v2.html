XIQClient.php — OAuth2 bearer token + APCu cache
Shared library at apdetail/includes/XIQClient.php. Namespace: Modules\APDetail\XIQClient. Token endpoint: POST https://api.extremecloudiq.com/oauth2/token. APCu cache keyed by client ID; filesystem fallback. Proactive refresh before expiry. On 401: clear cache, re-auth once, retry; surface error to view if second attempt fails. Parse RateLimit-Remaining header on every response — surface widget-level warning if <500. Credentials from Zabbix global macros {$XIQ_CLIENT_ID} / {$XIQ_CLIENT_SECRET}.

XIQ: device identity & config (/devices/{id})
Implement XIQClient::getDevice(int $id): array. Returns model, serial, firmware+build string (software_version — confirm single field vs two), config type, Mgt0 IP, MAC, NTP, last config push timestamp, network policy ID. 5-min APCu cache — config-class data; not per-page-load.
XIQ API S
XIQ: network policy + SSID list (/network-policies/{id}, /ssids/{id})
Implement XIQClient::getNetworkPolicy(int $id) → policy name + SSID ID array. Implement XIQClient::getSsid(int $id) → VLAN, auth method, encryption, band steering, enabled bands. Fan out one getSsid() call per SSID ID using curl_multi. 5-min cache per SSID ID. These are the slowest calls by volume — fan-out pattern is critical.

Map radio index to 2.4 GHz / 5 GHz band labels. No cache — called per page load for live telemetry. Point-in-time only — drives current-value badges for ch_util and noise on the Live Telemetry strip.
⚠ Endpoint path corrected from plan v1. No /xapi/v1/monitor/ prefix. Exact field names TBD pending live validation — placeholder keys documented in M0 XIQ Wireless Reference.
