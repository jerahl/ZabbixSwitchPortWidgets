3 LLD Discovery Rules — Template Items Requiring Prototypes

Discovery key decision: These use Zabbix SNMP LLD rules walking the .26928 enterprise table. The discovery rule OID walk must enumerate radio ifIndices (12, 13) and SSID subinterface ifIndices (17–23). SNMP index = ifIndex for all prototypes. Do not use a Zabbix built-in net.if.discovery key — that would bring in all interfaces including tunnels.
Discovery Rule A — Radio Interfaces (wifi0, wifi1)
Prototype itemOIDMacrosPoll intervalHistoryItem type
Radio Channel .26928.1.1.1.2.1.5.1.1.{#SNMPINDEX} {#IFINDEX} 5 min No (config) SNMP agent
TX Power (dBm) .26928.1.1.1.2.1.5.1.2.{#SNMPINDEX} {#IFINDEX} 5 min YES SNMP agent
Channel Width (encoded) .26928.1.1.1.2.1.5.1.3.{#SNMPINDEX} {#IFINDEX} 5 min No SNMP agent
Per-radio TX packets .26928.1.1.1.2.1.3.1.1.{#SNMPINDEX} {#IFINDEX} 1 min YES SNMP agent
Per-radio RX packets .26928.1.1.1.2.1.3.1.2.{#SNMPINDEX} {#IFINDEX} 1 min YES SNMP agent
Per-radio TX retries .26928.1.1.1.2.1.3.1.5.{#SNMPINDEX} {#IFINDEX} 1 min YES SNMP agent

⚠ Do NOT assume radio 0 = 2.4 GHz. The pilot AP runs dual-5G (channels 40 and 149). Label radio cards from the channel number: channels 1–14 = 2.4 GHz, ≥36 = 5 GHz. The Wireless tab must derive band from the channel value, never hardcode it.

Discovery Rule B — SSID Subinterfaces (ifIndex 17–23)
Prototype itemOID columnNotesHistory
SSID Name .26928.1.1.1.2.1.1.1.2.{#SNMPINDEX} STRING — the broadcast SSID name No (config)
SSID Interface Name .26928.1.1.1.2.1.1.1.1.{#SNMPINDEX} STRING — e.g. "wifi0.1" — maps SSID to radio No (config)
SSID TX bytes .26928.1.1.1.2.1.4.1.N.{#SNMPINDEX} Per-SSID traffic counter (col index TBD — validate during M1) YES

The SSID-to-radio mapping is derived from the interface name string (e.g. "wifi0.1" → radio wifi0). The SSID Broadcast table on the Wireless tab is primarily driven by XIQ data; the Zabbix SSID items serve as a fallback label source only.

4 Template Action Plan — What Needs to Change Before M2

1
Add CPU & Memory scalar items (enterprise OID)
The standard Extreme AP template likely uses HOST-RESOURCES-MIB for CPU/mem — which is absent on AP305C. Replace or add items using the confirmed enterprise OIDs.

CPU: SNMP agent · OID .1.3.6.1.4.1.26928.1.2.4.0 · key: extremeap.cpu.util · type: float · units: % · poll: 1 min · history: 90d
Memory: SNMP agent · OID .1.3.6.1.4.1.26928.1.2.10.0 · key: extremeap.mem.util · type: float · units: % · poll: 1 min · history: 90d

Add triggers: CPU >80% for 5 min → Average; Memory >85% for 5 min → Average.
Missing
2
Add Noise Floor master item + two dependent items
The noise floor string OID returns both radios in one combined string. Requires one master SNMP item and two dependent items with JS preprocessing to extract each radio's dBm value as a signed integer.

Master: SNMP agent · OID .1.3.6.1.4.1.26928.1.3.1.0 · key: extremeap.noise.raw · type: text · poll: 1 min · do not store history
Dependent wifi0: key: extremeap.noise.wifi0 · type: float · units: dBm · history: 90d
Preprocessing: JavaScript → parse string, return radio-0 integer value
Dependent wifi1: key: extremeap.noise.wifi1 · type: float · units: dBm · history: 90d

Threshold triggers: noise > −75 dBm → Warning; noise > −65 dBm → High.
Missing
3
Fix uplink traffic items — switch from ifHighSpeed to ifHCOctets
ifHighSpeed returns 0 on all AP305C interfaces. Any existing template items using net.if.in[ifHighSpeed.N] or net.if.out[ifHighSpeed.N] must be replaced.

Uplink In: SNMP agent · OID .1.3.6.1.2.1.31.1.1.1.6.10 (ifHCInOctets.10) · key: net.if.in[ifHCInOctets.10] · type: float · delta: per-second · units: bps · history: 90d · poll: 1 min
Uplink Out: SNMP agent · OID .1.3.6.1.2.1.31.1.1.1.10.10 (ifHCOutOctets.10) · key: net.if.out[ifHCOutOctets.10] · type: float · delta: per-second · units: bps · history: 90d · poll: 1 min

Note: ifIndex 10 = eth0 (physical uplink). Confirm ifIndex mapping holds fleet-wide — it was consistent across both walk APs but verify during M1 onboarding of new hosts.
Fix existing
4
Add LLD rule for Radio interfaces (.26928 radio table)
Walk .1.3.6.1.4.1.26928.1.1.1.2.1.5.1.1 to discover radio indices (returns ifIndex 12 and 13 for wifi0/wifi1). Create prototypes for: Channel, TX Power, Channel Width, per-radio error counters, per-radio retry counter.

Discovery filter: accept indices where the channel value is >0 (excludes any radio that's administratively down). Macro: {#RADIOIDX} = ifIndex, {#RADIONAME} = interface name string from col 1.

Poll channel and TX power at 5 min (config-class, changes rarely). Poll retry/error counters at 1 min (traffic-class, needed for sparklines).
Add LLD rule
5
Add LLD rule for SSID subinterfaces
Walk .1.3.6.1.4.1.26928.1.1.1.2.1.1.1.2 (SSID name column) to discover SSID indices (returns ifIndex 17–23 for BHS-56-Hallway's 7 configured SSIDs). Create prototypes for SSID name string and interface name (which encodes which radio the SSID runs on).

Primary purpose: populate the SSID name lookup table used in the PHP controller. XIQ drives the Wireless tab SSID table — this Zabbix item is the fallback when XIQ is unavailable.

Discovery filter: accept all non-empty SSID name strings.
Add LLD rule

Add Radio interface error items for wifi0 (ifIndex 12) and wifi1 (ifIndex 13)
These are standard IF-MIB counter items for the radio interfaces. They provide a radio health signal independent of XIQ.

ifInErrors.12 → net.if.errors[ifInErrors.12] · Counter · delta per-second · history 90d
ifInErrors.13 → net.if.errors[ifInErrors.13] · Counter · delta per-second · history 90d

Confirm and document LLDP item key patterns

No changes needed if the LLDP walk already exists. Just document the confirmed key names here.

Remove / disable standard HOST-RESOURCES-MIB items if present
The base Extreme AP template may include CPU/memory items using vm.memory.size[total], system.cpu.util against HOST-RESOURCES-MIB paths. These will produce "No Such Object" SNMP errors on AP305C and pollute Zabbix's unsupported item list.

Action: disable these items in the AP template (not delete — in case a future firmware adds the MIB). Replace with the enterprise OID items from Action 1.

6 Template Item Key Cheat-Sheet — PHP Controller Reference

These are the exact item key patterns the WidgetView.php controller must pass to API::Item()->get(). Use these strings verbatim in the filter array.

// Scalar items — single value, no discovery index
$scalar_keys = [
    'extremeap.cpu.util',          // CPU % · OID .26928.1.2.4.0
    'extremeap.mem.util',          // Memory % · OID .26928.1.2.10.0
    'extremeap.noise.wifi0',       // Noise floor radio 0 (dBm) — dependent item
    'extremeap.noise.wifi1',       // Noise floor radio 1 (dBm) — dependent item
    'extremeap.serial.0',          // Serial number string
    'extremeap.firmware.0',        // Firmware version string
    'system.uptime',               // Uptime ticks
    'net.if.in[ifHCInOctets.10]',  // Uplink in bps (NOT ifHighSpeed)
    'net.if.out[ifHCOutOctets.10]',// Uplink out bps
    'net.if.status[ifOperStatus.10]',// eth0 oper state
    'icmppingsec[{HOST.IP}]',      // ICMP latency (macro resolved by Zabbix)
    'icmppingloss[{HOST.IP}]',     // ICMP packet loss
];

// LLD prototype items — match with search['key_'] LIKE pattern
$lld_patterns = [
    'extremeap.channel.*',         // Radio channel — all discovered radios
    'extremeap.txpower.*',          // TX power dBm — all discovered radios
    'extremeap.ssid.name.*',        // SSID name — all discovered SSIDs
    'net.if.errors[ifInErrors.*]',  // Radio error counters
    'lldp.neighbor.sysname.*',      // LLDP neighbor switch name
    'lldp.neighbor.portid.*',       // LLDP neighbor port ID
    'snmp.interfaces.poe.dstatus.*',// PoE draw/class
];
Macro resolution note The icmppingsec and icmppingloss keys contain the {HOST.IP} macro as part of the item key as-stored in Zabbix. Use the search['key_'] LIKE filter with icmppingsec% and icmppingloss% rather than exact-match, because the macro is resolved to the literal IP in the stored item key (e.g. icmppingsec[172.16.97.59]).
