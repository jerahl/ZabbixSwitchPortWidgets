! Critical Corrections to M0 Assumed Paths

The endpoint paths in the project plan are WRONG. The plan assumed /xapi/v1/monitor/devices/{id}/wifi-data and /xapi/v1/monitor/devices/{id}/clients. The live OpenAPI spec (v25.9.0, April 2026) has no /xapi/ prefix and no /monitor/ path tier. The correct base URL is https://api.extremecloudiq.com and endpoints use version-less or versioned paths directly. Specific corrections are documented per endpoint below.
Client data is NOT a per-device endpoint. The /clients endpoint is a global query filtered by parameters (device ID, MAC, location, SSID, etc.), not /devices/{id}/clients. Confirm the correct filter parameter against the live AP during M0 validation.
"Get device WiFi interface stats" DOES exist. This is confirmed in the live API reference ToC under the Device section. It is the closest match to the planned wifi-data endpoint. Exact path and response schema require live validation against BHS-56-Hallway (device ID to be looked up via GET /devices search by serial or hostname).

1 Authentication

Two auth flows are available. Use API Token for the widget — it avoids credential storage and has an independently revocable key.

POST /login Confirmed
Username + password login. Returns a short-lived JWT bearer token. Token lifetime is approximately 2 hours based on XIQ Controller docs ("expires_in": 7200). Not recommended for widgets — use the API Token flow below.
// Request
{
  "username": "apiuser@tcs.k12.al.us",
  "password": "…"
}

// Response — 200 OK
{
  "access_token": "eyJhbGci…",  // JWT, ~2h lifetime
  "token_type":   "Bearer"
}
POST /auth/apitoken Confirmed
Creates a long-lived API access token. This is the correct credential type for the widget. Store the generated token in a Zabbix global macro ({$XIQ_API_TOKEN}). Token does not expire on a fixed timer but can be individually revoked in the XIQ Administration → API Access Tokens UI.
// Request (requires existing bearer token to authorize creation)
// Authorization: Bearer <login_token>
{
  "description": "Zabbix AP Dashboard widget"
}

// Response — 200 OK
{
  "token": "…"  // store in {$XIQ_API_TOKEN} — shown once
}
Widget auth strategy: Generate the API Token once via the XIQ web UI or a one-time CLI script. Store it in a Zabbix global macro. In XIQClient.php, send it as Authorization: Bearer {$XIQ_API_TOKEN} on every request — no token refresh loop required. On 401, surface an error (the token has likely been revoked) rather than attempting re-auth with stored credentials.

4 Answer: Does XIQ Expose Historical Time-Series? — VERDICT: NO

XIQ is point-in-time only for all wireless monitoring metrics. The API exposes current snapshots, not time-series history. There are no start_time / end_time parameters on the wifi-stats or client endpoints. XIQ is a cloud management and configuration platform — it does not function as a TSDB. Historical wireless telemetry for sparklines must come from Zabbix calculated items only.

7 Rate Limits & Polling Budget

Confirmed from developer.extremecloudiq.com: 7,500 API requests per hour per customer (VIQ), sliding window algorithm. Quota is shared across ALL users and integrations in the same XIQ account. Response headers: RateLimit-Limit: 7500;w=3600 · RateLimit-Remaining: N · RateLimit-Reset: S. HTTP 429 when exceeded. Contact Extreme support to increase quota.

~8 XIQ calls per dashboard refresh. At a 60-second auto-refresh with 10 operator sessions open simultaneously: 10 × 8 = 80 requests/min = 4,800/hour. Well within the 7,500/hour quota but leaves limited headroom. Session-level APCu caching is required — each Zabbix web worker must share a token and config cache, not issue fresh calls per PHP worker process.

XIQ quota is shared with all other XIQ API consumers in the TCS VIQ account. If TCS uses other XIQ integrations (SolarWinds NPM is confirmed to consume XIQ API), budget accordingly. Monitor RateLimit-Remaining headers in production and surface a widget-level warning if remaining quota drops below 500. Contact Extreme support to increase the default quota before deploying to multiple schools beyond BHS pilot.
